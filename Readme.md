# Git & DevOps Skills: LPI 701-100 Preparation

This repository serves as a practical workspace for mastering **Git** and **DevOps** workflows. The primary objective is to build hands-on proficiency in preparation for the **LPI DevOps Tools Engineer (701-100)** certification exam.

## 🎯 Purpose
The goal of this project is to:
* Practice advanced Git operations (branching, merging, rebasing).
* Implement CI/CD concepts locally and via GitHub Actions.
* Document configuration management and automation scripts.
* Validate knowledge against the official LPI 701-100 objectives.

---

## 📚 LPI DevOps Tools Engineer (701-100) Exam Objectives

### Topic 701: Software Engineering
- [x] **701.1 Modern Software Development:** Concepts of Agile, Lean, and CI/CD.
- [x] **701.2 Standard Components and Platforms:** Microservices, APIs, and cloud-native design.
- [x] **701.3 Source Code Management:** Advanced Git usage, branching strategies, and repository management.
- [] **701.4 Continuous Integration and Continuous Delivery:** Building pipelines and automated testing.

### Topic 702: Container Management
[] **702.1 Container Usage:** Docker fundamentals, image creation, and registry management.
[] **702.2 Container Orchestration:** Kubernetes basics and container networking.
[] **702.3 Container Infrastructure:** Storage and security for containerized environments.

### Topic 703: Machine Management
[] **703.1 Virtualization:** Managing VMs and Vagrant.
[] **703.2 Cloud Computing:** IaaS, PaaS, and SaaS concepts.
[] **703.3 Configuration Management:** Ansible, Chef, or Puppet automation.

### Topic 704: Service Operations
[] **704.1 Monitoring and Logging:** Prometheus, Grafana, and ELK stack.
[] **704.2 Log Management:** Centralized logging and analysis.
